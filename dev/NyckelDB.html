init(tableHeaders, columnPropertiesopt, optionsopt, callbackopt)

Initiate the database. Required to call this right after creating a new NyckelDB obj. Sets initial values, checks for cached data and builds the search index

Parameters:
Name Type Attributes Description
tableHeaders Array.<string> | object

an array of the names of all of the columns, or an object containing column header names and properties
columnProperties Array.<string> | object <optional>

optional, if not specified in with tableHeaders, as an array if just types, or an object if also other column properties
options object <optional>

{ customProperties: { : { initialValue: , type: "String", "Boolean" or "Number" }, ... }, doNotIndex: , any columns that do not need to ever be search indexed. Can speed up load times initialIndex: , columns to be search indexed on load. Specifying this value can speed up load times key: , token: importData: , table data that is ready to drop in to the database without parsing importJSON: , json data that needs to be parsed importCSV: , CSV data that needs to be converted to JSON and then parsed }
callback successCallback <optional>
